# 声で数を読む練習アプリ「数読み（kazu-yomi）」設計書

作成日: 2026-06-15

## 1. 何のアプリか（一言で）

3桁の数字を、**位を意識して声で正しく読む**練習アプリ。
画面に出た「365」を「さんびゃく ろくじゅう ご」と声で読み、機械が聞き取って○×を判定する。

### ねらい（背骨）
人間の先生は「だいたい合ってる」を優しく汲み取ってくれる。でも機械は汲み取らない。
**「ちゃんとはっきり、正しく言わないと通らない」** という体験そのものを教材の力にする。
"汲み取らなさ" を欠点ではなく、子どもへの良い強制力に変える。
（ぽこぴぃの「困る手前ですくう」と地続き）

### 対象
小学2年生（百の位を習う学年）

## 2. 画面（1枚もの）

```
┌─────────────────────────────┐
│  百の位  │  十の位  │  一の位   │  ← 位の表（ラベル）
│    3    │    6    │    5     │  ← 数字（常に表示）
│─────────┼─────────┼──────────│
│  🪙×3   │  🪙×6   │  🪙×5    │  ← お金ヒント（オン/オフ切替）
│         │         │          │
│        🎤 マイクボタン         │  ← 押して読む
└─────────────────────────────┘
```

- 上：位の表（百／十／一）＋大きな数字
- 下：お金ヒント（100円玉・10円玉・1円玉を枚数ぶん）※オン/オフ切替
- 中央：🎤マイクボタン → 押して声で読む
- 結果：○ or ×

## 3. 段階（難しさ）

| 段階 | 数字 | お金 | やること |
|---|:---:|:---:|---|
| Lv.1 | ✅ | ✅ | 両方見て読む（やさしい） |
| Lv.2 | ✅ | ❌ | 数字だけ見て読む（むずかしい・最終目標） |

- 数字は常に表示。**お金ヒントのオン/オフだけ**で難しさが変わる。
- お金は「位の見える化」＝補助輪。最終的には外して数字だけで読めるのが目標。

## 4. 出題する数字

- **フェーズA（まず作る）**: 0なし（111〜999、各位1以上）
- **フェーズB（確実に足す）**: 0あり（108, 250, 300 など "位が抜ける" パターン）
  - 0ありこそ位の読みの核心（「ひゃくぜろはち」と言ってしまう所を矯正できる）
  - お金ヒントでは、0の位を**「空っぽの皿」**として見せる → 「ここは読まない」が目で分かる
- まずAで形を完成・動作確認 → そのうえでBへ拡張する

### 読みの正解生成（実装メモ）
位ごとの正しい読みを自動生成する。百の位の音変化に注意：
- 300 = さん**びゃく** / 600 = ろ**っぴゃく** / 800 = は**っぴゃく**
- 100 = ひゃく（「いちひゃく」ではない）
- 十の位：10 = じゅう（「いちじゅう」ではない）

## 5. 判定（このアプリの山場）

**方針：きびしめ。位で正しく読めたときだけ○。**

- 「さん ろく ご」のような1桁バラ読みは、狙いと別物なので○にしない。
- 判定はアプリ内で一瞬（サーバー往復なし）。

### 正直な技術的限界（合意済み）
声を文字にする機械は、「さんびゃくろくじゅうご」も「さんろくご」も
**両方 "365" と文字にしてしまう瞬間**がどうしても残る。
→ 「1桁読みを100%はじく」とは約束できない。
**はっきり位で読めた時は○／あいまい・バラバラな時は×** という挙動になる。
これは狙い（はっきり言わないと通らない体験）とむしろ相性が良い。
実際に動かして厳しさのさじ加減を一緒に調整する。

## 6. ×（うまくいかない）ときの動き

1. **もう一回いえる**（リトライボタン）
2. **正しい読みをお手本で聞かせる**（ブラウザの読み上げで「さんびゃく…ろくじゅう…ご」）
3. **お金ヒントを自動でオンに戻す**（足場を戻して助ける）

## 7. お金イラストの並べ方（既存アプリ踏襲）

- 画像素材：`money_100.png` / `money_10.png` / `money_1.png`（いらすとや・1枚ずつ）
  - アプリが必要な枚数だけ並べて描く（7枚なら7個コピー表示）
- 並べ方ルール（「りんごの数えかた」アプリと同じ）：
  - 各位ごとに**縦に積む（1列5枚まで）→ 6枚目から右にもう1列**
  - **左の列から埋める／左寄せ／5枚以下でも中央に寄せない**（左列固定）
  - 個数が変わっても**各コインの位置はずれない**（位置固定）
  - → 既存アプリ（howmanyapples 等）の配置ロジックを**忠実にコピー**する

## 8. 体験（速さ）

- 言い終わって**約1秒で○×**が出るサクサク感を目標。
- 声の聞き取り＝ブラウザ標準のリアルタイム認識。
- お手本の声＝ブラウザのその場読み上げ（音声ファイル読込なし）。

## 9. 作り（インフラ）

- 今ある算数アプリ群（NUMBER GATE / ものさし等）と同じブラウザアプリ。
- ホスティング：Cloudflare（追加費用ほぼゼロ）。
- 個人情報なし・使い捨て（ログイン不要、教材アプリ扱い）。
- 声の聞き取りは裏でネット経由（Google の仕組み）を通るが、**保存は一切しない／読むのは数字だけ**。

### 正直メモ
ブラウザ標準の音声認識は **Chrome 前提**（Safari は非対応/不安定）。
→ 推奨ブラウザを Chrome に固定する。

## 10. やらないこと（YAGNI）

- お金だけ（数字オフ）の段階＝作らない
- 4桁以上・小数＝今回は対象外
- ランキング・ログイン・記録保存＝なし

## 11. 残課題（実装フェーズで詰める）

- 「位読み」と「バラ読み」を見分けるロジックの具体（スペース有無などのヒューリスティック）
- 厳しさのさじ加減（実機テストで調整）
- 0ありフェーズBのお金「空っぽの皿」の見せ方の最終ビジュアル
